## Convention notes (re-read before each phase)

- **Per-app folder for SPA:** `apps/web/src/{api,auth,routes,fragments,components,hooks,stores,lib,styles}/`. Tests under `apps/web/tests/{unit,e2e}/`.
- **No shared SPA package.** Routes/components live inside `apps/web` only. Helpers shared with backend (e.g. risk color scale) get a small re-export in `lib/`, never a cross-app import.
- **API client structure:** `src/api/<service>.ts` exports named functions, never a class. `src/api/index.ts` re-exports them as `api.<service>.<method>(...)`.
- **Contracts:** import from `@corlens/contracts/<service>` with namespace alias `cc / pp / ag / id` (corridor / path / agent / identity) â€” same convention as backend.
- **Lazy routes:** every route is `lazy()` in `App.tsx` (v1 already does this â€” keep it).
- **Tests:** Vitest with `jsdom` env for component tests, native Node for pure logic. RTL (`@testing-library/react`) for component tests.
- **Per-commit discipline:** `pnpm biome check` clean, `pnpm -r typecheck` clean, all touched Vitest suites green, `pnpm --filter @corlens/web build` clean. No `--no-verify`.

- **`@crossmarkio/sdk` v0.4 API quirks.** If the sign-message API differs from v1 expectations, fall back to manual challenge concatenation matching `apps/identity/src/services/auth.service.ts` verification.
- **POST-as-SSE for SafePath.** `EventSource` only supports GET. Implementation uses `fetch` + `ReadableStream`. If browser compatibility issues appear, wrap with a small polyfill check (Chrome/Firefox/Safari all support ReadableStream as of 2023).
- **PDF download with localStorage JWT.** v1 relied on cookie auth fallback. For v2, fetch + blob URL works but needs explicit Auth header. If file size > 50 MB this becomes memory-heavy; cap PDF size server-side (already done, ~2 MB max).
- **Bundle size.** If cobe + three + reactflow + react-markdown push above 1.2 MB compressed, the Landing route is already lazy-loaded; consider moving CorridorGlobe behind a `<Suspense>` boundary too.
