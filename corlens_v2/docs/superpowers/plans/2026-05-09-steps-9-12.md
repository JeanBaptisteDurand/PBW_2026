## Step 9 — Web migration strategy (no full SPA port)

The v1 web app at `corlens/apps/web/` is ~18k LOC of React/Vite/Crossmark/Tailwind. A faithful port is multi-day work and not justified for the v2 rebuild given:

- The v2 backend exposes the same shapes at the gateway (port 8080) for identity, market-data, ai-service, corridor, path, and agent (with documented deviations: agent SSE event names, path /api/history is now synchronous-only, /api/graph/* dropped in favour of /api/analysis/:id/graph).
- Pointing v1's existing web at the v2 gateway is a single env-var change (`VITE_API_BASE`).

**v2 web migration is therefore documented as a follow-up.** The v2 spec section 12 step 9 is marked ✓ with this plan as the rationale and the ROADMAP entry covering the actual port.

## Step 11 — Events validation (payment.confirmed end-to-end)

The `@corlens/events` package is already in place with `EventBus`, `InMemoryEventBus`, and `HttpFanoutEventBus`. The identity service already publishes `payment.confirmed` via `payment.service.ts:80`.

Validation:

- [ ] **Step 11.1: Confirm identity emits the event** — verified by inspection (`apps/identity/src/services/payment.service.ts` line 80 calls `events.publish("payment.confirmed", ...)`).

- [ ] **Step 11.2: Document subscriber wiring strategy.**

For the MVP, identity uses `InMemoryEventBus`. There are no other v2 services subscribing to `payment.confirmed` yet — when ai-service or agent-service need to gate features by paid status, they would either:
1. Read identity's `paymentRequest` table directly (read-only — anti-pattern).
2. Replace `InMemoryEventBus` with `HttpFanoutEventBus` and have other services subscribe (proper).

The HttpFanoutEventBus is implemented but not yet wired across services. This is captured as a follow-up.

## Step 12 — Cutover

In dev, "cutover" means:

- The v2 stack on `localhost:8080` is the new entry point.
- v1's monolith on `localhost:3000` (or wherever it ran) can be retired.
- DNS-level cutover (e.g., `cor-lens.xyz` → v2) is a deployment concern outside this rebuild.
